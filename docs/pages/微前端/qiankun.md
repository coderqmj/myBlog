### 1.基本概念

- 较早的微前端框架，建立在single-spa基础上
- 主要做两件事情：
  - 加载资源
  - 资源隔离
- 优点：
  - 技术栈无关，不同子项目使用不同技术栈
  - 解耦，微前端技术将大项目拆分为各个子应用，

### 2.qiankun的原理

```
1.应用加载：qiankun动态创建script标签加载子应用，加载JS会执行子应用暴露的生命周期函数
2.生命周期函数：每个子应用都需要暴露出对应的bootstrap（加载），mount（挂载），unmount（卸载）三个生命周期函数
3.JS沙箱隔离：Proxy 对象创建了一个JS沙箱，用于隔离子应用的全局变量，防止变量污染
4.样式隔离：触发子应用添加样式标签，卸载时移除
5.通信机制：qiankun 提供了一个全局的通信机制，允许子应用之间进行通信。
```
