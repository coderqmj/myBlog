### 1.使用Figma Mcp提效

```
1.一个大型项目，之前还原UI样式稿，效率低下，即使开发的时候仔细对比，开发完了也自测对比样式稿，仍然有些细节经常会漏掉，比如间距之类的，字体大小等等。这种情况，UI需要走查一天，Web这边修复样式走查也需要一天，而且经常会出现测试OK了，却因为UI样式有偏差导致无法发布。既影响UI和前端的效率，也耽误发布计划。有了Figma Mcp+Cursor之后大大提高了效率，针对于UI部分，起码提升效率80%+。走查+修复UI基本上几个小时就搞定了。
2.首先是申请figma的开发者模式，开发模式和视察模式不一样的是，开发者模式的样式稿里面有实现好的样式代码，包括变量还有Icon之类的。
3.然后就是在Cursor中安装Figma Mcp，把url传进去就好。
4.然后就是在UI稿中复制选中的元素，是一个连接link，到cursor里面复制起来，让cursor使用tailwind实现

它做的事情：
1.首先会去读取mcp的rules，比如说你的模型，当前是什么语言（js，css），然后是什么框架啊，React。
2.再去生成你选中的模块的样式
```
