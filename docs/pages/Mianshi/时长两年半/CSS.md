### 1.如何做样式隔离

```
Q：为什么做样式隔离？
	1.CSS选择器的作用域是全局的，所以很容易引起选择器冲突；写很长的类名导致复杂度上升
	2.CSS缺少抽象机制，选择器很容易出现重复，不利于维护和复用
	3.总结：污染、可读性、阅读性
Q：样式隔离方案？
	1.BEM，一种CSS命名规则，Block Element Modifier的缩写，通常用于组件库
		示例：.dropdown-menu_item--active
		优点：开发者严格遵守BEM规范，可以解决样式污染问题，可读性也好，一看类名就知道哪个DOM
		缺点：编写的样式名称太长了
	2.CSS modules
		原理是在对应组件中，导入css作为style毒性，通过style点属性，赋值给dom样式使用
		需要配置css-loader
		打包编译后类名会变成独一无二的哈希，css和js文件都会
		优点：解决样式污染
		缺点：编译后的文件可读性几乎没有
	3.css in js
		比如styled-components，像写js一样
		优点：
			1.解决样式污染
			2.没有样式堆积问题，css in js是和组件绑定在一起的，组件不用删除的时候，无需考虑可以直接清理该样式，不会对其他组件造成影响
			3.CSS in js直接将样式卸载js文件里面，所以样式复用以及逻辑判断都十分方便
	4.预处理器
		less、sass等，这些都支持：代码混合、嵌套选择器、集成选择器，这些特性让css文件有更好的阅读性和可读性
		优点：
			1.利用嵌套，可以解决样式污染问题
			2.可读性好，一目了然哪个dom，对于无用的css，删除dom后，对应的css可以放心删除
		缺点：
			1.需要借助相关编译工具处理
```
